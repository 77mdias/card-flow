# Backlog

Este diretorio concentra tudo que ainda nao foi priorizado para sprint.

Pode incluir:
- ideias de produto
- requisitos novos
- melhorias tecnicas
- sugestoes de UX
- exploracoes de arquitetura

Regra de fluxo:
1. Item nasce no backlog.
2. Item triado vira task-spec.
3. Item priorizado entra em `sprints/`.
