# Sprint 02 - Cartoes + Statements Base

## Objetivo
Permitir gestao de cartoes e preparar estrutura de statements.

## Escopo
- CRUD de cartoes com validacoes.
- regras de `closing_day` e `due_day`.
- modelo inicial de statement por periodo.

## Criterios de aceite
- cartao criado/editado/excluido com validacao.
- nenhum dado sensivel de cartao persistido.
- statement por periodo consultavel.

## Fora de escopo
- regra completa de transacao automatica.
